# LRC-Maker AI 辅助对齐（本地计算引擎）

基于 Whisper 语音识别模型的本地歌词打轴引擎，配合 [LRC-Maker](https://lrc-maker.github.io/) 的波形图界面使用，可实现“AI 自动对齐 + 网页可视化微调 + KTV 逐字动感预览”的工作流。它适合需要快速生成初稿、再进行精细校准的歌词制作场景。

> **新手先看这里**
> 这个项目由两部分组成：
> 1. 你需要先在电脑上启动本仓库提供的 **本地 AI 引擎**。
> 2. 然后在浏览器里安装 **Tampermonkey** 和对应脚本。
>
> 网页本身不负责运行 AI，只有两部分配合后才能正常使用。

## 快速开始

### 1. 下载并启动本地 AI 引擎
本地引擎负责实际的语音识别与对齐，v2.0 已经**内置了全部运行环境与 AI 模型**，不需要配置 Python，解压即是完全离线版。

1. 前往 [Releases](https://github.com/Flare-Sky/LRCMaker-AI-Backend/releases) 页面下载对应平台的压缩包。
2. 解压到一个方便找到的位置，建议路径中不要包含中文字符。
3. 根据系统运行对应程序：
   * **Windows**：下载 `-Windows-v2.0.zip`，解压后双击运行 `.exe` 程序。
   * **Mac (Apple 芯片)**：下载 `-Mac-Arm64-v2.0.zip`，解压后双击黑色的可执行文件。
   * **Mac (Intel 芯片)**：下载 `-Mac-Intel-v2.0.zip`，解压后双击黑色的可执行文件。
4. 当黑框终端窗口显示 `✅ AI 引擎就绪！` 时，说明启动成功。（*注：如果默认的 8000 端口被占用，程序会自动顺延启用新端口并在黑框中醒目提示，请留意日志。*）

#### Mac 用户额外说明
由于程序未进行苹果官方高昂的开发者签名，首次运行时可能会被系统安全策略拦截（提示文件损坏或无法验证开发者）。若出现加载失败或闪退，请先在终端中执行以下操作：

1. 打开 Mac 自带的 Terminal（终端）。
2. 输入 `xattr -cr `，注意末尾保留一个空格。
3. 将解压后的文件夹整个拖入终端窗口，补全路径后按回车。
4. 重新双击运行程序即可。

### 2. 安装浏览器插件管理器
为了让网页与本地引擎通信，需要先安装脚本管理器。

* 推荐使用 Edge 或 Chrome 浏览器。
* 安装 [Tampermonkey（油猴插件）](https://www.tampermonkey.net/)。

### 3. 安装专属脚本
* 打开 [Greasy Fork 安装页](https://greasyfork.org/zh-CN/scripts/579199-lrc-maker-ai-%E8%BE%85%E5%8A%A9%E5%AF%B9%E9%BD%90-%E9%85%8D%E5%90%88%E6%9C%AC%E5%9C%B0%E5%BC%95%E6%93%8E) 并更新至最新的 **v2.0** 版本。
* 安装完成后，浏览器侧的准备工作就结束了。

## 使用方式

1. 确保本地 AI 引擎窗口处于运行状态。
2. 打开 [LRC-Maker 官网](https://lrc-maker.github.io/)。
3. 页面左下角会出现 **AI LRC 生成器** 面板，可按 `Alt + A` 快捷键显示或隐藏。
4. 在左侧点击 **加载音频**，导入歌曲文件。
5. 在 AI 面板中粘贴纯文本歌词，工具会自动处理作词、作曲等非演唱内容。
6. **(全新特性)** 选择你需要的输出模式：「普通单句 LRC」或「逐字动感 KTV」。
7. 点击 **🚀 一键 AI 强制对齐**。
8. 等待处理完成后，歌词会自动写入编辑区。如果你选择了 KTV 模式，可以直接在面板上方的视窗中体验原生渲染的逐字高亮预览！

## 常见问题

**Q:点击对齐后提示“生成失败 / 请检查本地 API 服务器是否启动”怎么办？**

A:首先检查本地引擎的黑框窗口是否仍在运行。其次，查看黑框里的启动日志，如果提示默认的 8000 端口被占用并切换到了 8001 或其他端口，**请务必在网页 AI 面板的配置中，将 API 端口同步修改为对应的新端口。**

**Q:歌词中的作词、作曲等信息会被当成歌词一起处理吗？**

A:不会。只要这些信息是类似 `作词：张三` 或 `作曲: 李四` 的格式，脚本会自动识别并将其视为非演唱内容。如果你想更明确地区分正文与说明，也可以在正片歌词前加一行 `---`，其上方内容会被统一视为说明信息。

**Q:第一次启动需要等待很久吗？还需要联网下载模型吗？**

A:完全不需要了！v2.0 满血版已经将约 500MB 的 Whisper `small` 模型和所需环境全部打包内置在了 `models` 文件夹中。解压后即可实现 100% 纯离线秒开，彻底告别国内网络环境导致的下载报错。

**Q:如果不再使用，如何清理本地文件和模型？**

A:v2.0 引擎实现了真正的“绿色便携”，不会向你的电脑深层系统目录（如 `~/.cache`）写入任何隐藏文件。如果你不想用了，只需将解压出来的整个文件夹拖入废纸篓/回收站删除即可，绝不残留垃圾。
